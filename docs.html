N.O.V.A.
Documentation

A fully local, voice and text controlled AI desktop assistant. Talk to it, type to it — NOVA understands intent, routes to the right module, and responds through voice and a live chat window.

What makes NOVA different
NOVA isn't a wrapper around a chatbot. It has a real intent routing system, persistent memory that summarizes sessions using AI, 13 functional modules, full GUI with system tray, and a first-time setup wizard. Built entirely in Python, runs locally.
Getting Started

Installation

1

Clone the repository

git clone https://github.com/shreyans-gits/voice-controlled-system
cd NOVA
2

Create a virtual environment (recommended)

python -m venv venv
venv\Scripts\activate
3

Install dependencies

pip install -r requirements.txt
4

Run the setup wizard

python setup.py

This opens a GUI where you enter your API keys and preferences. It generates your .env file automatically and then closes itself.

5

Launch NOVA

python nova.py
Windows Only
NOVA is built for Windows. System control (volume, brightness), pycaw audio, and system tray features are Windows-specific. Linux/macOS support is not guaranteed.

Architecture

How It Works

→You speak or type a query
→voice.listen() transcribes speech via Google Speech API, or text input is picked up from the GUI
→brain.get_intent() sends the query to Groq — returns one word like WEATHER or CONVERSATION
→If needed, brain.extract_subject() pulls the clean subject out — "play some Drake" → Drake
→The correct module is called with the subject
→Result is displayed in the chat window and spoken aloud
→If intent is CONVERSATION, the exchange is logged to memory
→On shutdown, Groq summarizes the session and merges it with past memory
→Next startup, the summary is injected into the system prompt — NOVA remembers you

Architecture

Memory System

NOVA has persistent memory across sessions. It doesn't store raw logs — it uses Groq to compress conversations into a meaningful summary of who you are.

→During a session, CONVERSATION exchanges are logged in RAM via memory.log()
→On shutdown, memory.summarize_and_save(brain) sends the log to Groq
→Groq extracts key facts and merges them with the previous summary
→Final summary saved to memory.json — under 150 words
→Next startup, summary injected into the system prompt so NOVA already knows you
Say "clear your memory"
This triggers the FORGET intent which calls memory.clear(), wiping both the session log and memory.json. You can also clear memory from the Settings window.
Modules

Core Modules

brain.py

The AI backbone. Handles intent classification, subject extraction, number extraction, conversation history, and session summarization. Uses Groq's Llama 3.3 70b model.

  • get_intent(query)
  • extract_subject(query, intent)
  • extract_number(query, intent)
  • ask(user_input)
  • summarize(log, old_summary)
memory.py

Manages persistent memory. Logs conversation exchanges during the session and compresses them into a summary on shutdown.

  • load() — reads memory.json
  • log(sender, text)
  • summarize_and_save(brain)
  • clear()
voice.py

Handles all audio I/O. Speech recognition via Google Speech API, TTS via Microsoft edge_tts with configurable voice and speed.

  • speak(text)
  • listen() → query string
  • get_speed() → reads settings.json
dashboard.py

The main GUI. CustomTkinter window with color-coded chat, animated waveform, status indicator, text input, and system tray integration.

  • add_message(sender, text)
  • set_status(value)
  • animate_waveform()
  • handle_input()

Contributing

Adding a new module to NOVA is straightforward. Every module follows the same pattern:

1

Create your module file

Add a new file in modules/. Create a class with an __init__ and the methods you need.

2

Add your intent to brain.py

Add your new intent word to the list inside get_intent()'s prompt so Groq knows it exists.

3

Import and initialize in nova.py

Import your module at the top and initialize it in the main() function alongside the others.

4

Add the intent handler

Add an elif intent == "YOUR_INTENT": block in the main loop in nova.py. Call your module, put the result in message_queue, and call voice.speak().
